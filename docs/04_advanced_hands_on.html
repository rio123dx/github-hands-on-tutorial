🔧 Phase 1: Windows環境の設定

⏱️ 所要時間： 約45分

1.1 WSL2のインストール

Windows Subsystem for Linux 2を有効化してLinux環境を構築します。

ステップ1: PowerShellを管理者権限で開く

  1. スタートメニューで「PowerShell」を検索
  2. 「Windows PowerShell」を右クリック → 「管理者として実行」
📸 スクリーンショット: PowerShell管理者実行

ステップ2: WSL機能を有効化

以下のコマンドを実行してWSLを有効化します：

PS C:\WINDOWS\system32> wsl --install # Ubuntu ディストリビューションがインストールされます
⚠️ 注意: インストール完了後、システムの再起動が必要です。再起動後、Ubuntu のセットアップ画面が自動で開きます。

ステップ3: Ubuntu セットアップ

  1. 再起動後、Ubuntu ターミナルが開きます
  2. ユーザー名を入力（半角英数字、推奨：自分の名前）
  3. パスワードを設定（入力時は表示されませんが入力されています）
# Ubuntu セットアップ画面の例 Enter new UNIX username: yourname New password: (パスワードを入力) Retype new password: (パスワードを再入力)

✅ チェックポイント

  • □ WSL2が正常にインストールされた
  • □ Ubuntu ターミナルが起動する
  • □ ユーザー名とパスワードが設定済み

1.2 Visual Studio Codeのセットアップ

プロ仕様の開発環境を構築します。

ステップ1: VS Code本体のインストール

  1. VS Code公式サイトからダウンロード
  2. インストーラーを実行
  3. 「PATHに追加」オプションを有効にして完了
📸 スクリーンショット: VS Codeインストール画面

ステップ2: 必須拡張機能のインストール

VS Codeを開いて、以下の拡張機能をインストールします：

  1. WSL拡張機能
    • 拡張機能タブ（Ctrl+Shift+X）を開く
    • 「WSL」で検索 → Microsoft製をインストール
  2. Japanese Language Pack
    • 「Japanese Language Pack for Visual Studio Code」をインストール
    • VS Code再起動後、日本語で表示される
  3. GitHub Copilot（オプション）
    • 「GitHub Copilot」をインストール
    • GitHubアカウントでサインイン
💡 プロのヒント: WSL拡張機能により、WindowsのVS CodeからLinux環境で直接開発できます。これが現代的な開発スタイルです。

✅ チェックポイント

  • □ VS Codeが正常にインストールされた
  • □ WSL拡張機能が有効になっている
  • □ VS Codeが日本語で表示されている
  • □ 左下に「WSL: Ubuntu」の表示がある

🔐 Phase 2: GitHub設定

⏱️ 所要時間： 約30分

2.1 SSH認証の設定

セキュアな認証方式でGitHubに接続する設定を行います。

ステップ1: SSH鍵ペアの生成

VS CodeでUbuntu ターミナルを開きます：

  1. VS Codeで Ctrl + Shift + ` を押してターミナルを開く
  2. 以下のコマンドを実行：
user@hostname:~$ ssh-keygen -t ed25519 -C "your-email@example.com" # your-email@example.com を実際のメールアドレスに変更 Generating public/private ed25519 key pair. Enter file in which to save the key (/home/user/.ssh/id_ed25519): (Enterを押下) Enter passphrase (empty for no passphrase): (Enterを押下 - パスフレーズなし) Enter same passphrase again: (Enterを押下)

ステップ2: 公開鍵をクリップボードにコピー

user@hostname:~$ cat ~/.ssh/id_ed25519.pub # 表示された内容をコピー (ssh-ed25519 から始まる長い文字列)

ステップ3: GitHubに公開鍵を登録

  1. ブラウザで GitHub SSH keys設定 を開く
  2. 「New SSH key」をクリック
  3. Title: 「WSL Ubuntu」
  4. Key: コピーした公開鍵を貼り付け
  5. 「Add SSH key」をクリック
📸 スクリーンショット: GitHub SSH key追加画面

ステップ4: SSH接続テスト

user@hostname:~$ ssh -T git@github.com The authenticity of host 'github.com' can't be established. ... Are you sure you want to continue connecting (yes/no/[fingerprint])? yes Hi [username]! You've successfully authenticated, but GitHub does not provide shell access.

✅ チェックポイント

  • □ SSH鍵ペアが生成された
  • □ 公開鍵がGitHubに登録された
  • □ SSH接続テストが成功した

2.2 Git設定の初期化

Gitの基本設定を行います。

ユーザー情報の設定

user@hostname:~$ git config --global user.name "Your Name" user@hostname:~$ git config --global user.email "your-email@example.com" user@hostname:~$ git config --global init.defaultBranch main # 設定確認 user@hostname:~$ git config --list

✅ チェックポイント

  • □ ユーザー名が設定された
  • □ メールアドレスが設定された
  • □ デフォルトブランチが main に設定された

🚀 Phase 3: 開発ワークフロー実践

⏱️ 所要時間： 約60分

3.1 リポジトリのクローンとセットアップ

ブラウザ版で作成したリポジトリをローカルにクローンします。

作業ディレクトリの作成

user@hostname:~$ mkdir ~/projects user@hostname:~$ cd ~/projects user@hostname:~/projects$ git clone git@github.com:[username]/js_tutorial.git user@hostname:~/projects$ cd js_tutorial

VS Codeでプロジェクトを開く

user@hostname:~/projects/js_tutorial$ code . # VS Code が開き、プロジェクトが表示されます
🎉 成功！ これで本格的な開発環境が整いました。VS CodeでGitHubリポジトリを直接編集できます。

✅ チェックポイント

  • □ リポジトリがローカルにクローンされた
  • □ VS Codeでプロジェクトが開いている
  • □ ファイル一覧が表示されている

3.2 ブランチ作成と機能開発

新機能を安全に開発するためのブランチ作成を体験します。

ステップ1: 新しい機能ブランチの作成

user@hostname:~/projects/js_tutorial$ git checkout -b feature/enhanced-todo # 「enhanced-todo」機能用のブランチを作成して切り替え user@hostname:~/projects/js_tutorial$ git branch # 現在のブランチを確認 main * feature/enhanced-todo

ステップ2: HTMLファイルの更新

VS Codeで index.html を開いて、v2.0の機能に対応させます：

  1. index.html の <script src="script.js"> を <script src="script_v2.js"> に変更
  2. ファイルを保存（Ctrl+S）

ステップ3: 新機能ファイルの作成

VS Codeで新しいファイル script_v3.js を作成：

// script_v3.js - 削除機能付きToDoアプリ
const taskList = document.getElementById('taskList');
const addButton = document.getElementById('addButton');
const taskInput = document.getElementById('taskInput');

// 初期データ
const initialTasks = [
'洗濯をする',
'掃除をする',
'買い物に行く'
];

// 初期データを表示
initialTasks.forEach(taskText => {
addTaskToList(taskText);
});

// タスクをリストに追加する関数
function addTaskToList(taskText) {
const listItem = document.createElement('li');
listItem.innerHTML = `
<span>${taskText}</span>
<button class="delete-btn" onclick="deleteTask(this)">削除</button>
`;
taskList.appendChild(listItem);
}

// 削除機能
function deleteTask(button) {
button.parentElement.remove();
}

// 追加ボタンの処理
addButton.addEventListener('click', () => {
const newTaskText = taskInput.value.trim();
if (newTaskText !== '') {
addTaskToList(newTaskText);
taskInput.value = '';
taskInput.focus();
}
});

// Enterキーでの追加
taskInput.addEventListener('keypress', (e) => {
if (e.key === 'Enter') {
addButton.click();
}
});

✅ チェックポイント

  • □ 新しいブランチが作成された
  • □ index.htmlが更新された
  • □ script_v3.jsが作成された
  • □ VS Code左側でファイル変更が表示されている

3.3 変更のコミットとプッシュ

VS Codeの統合Git機能を使って変更をコミットします。

ステップ1: VS CodeのGit機能を使用

  1. VS Code左側の「ソース管理」タブ（Git アイコン）をクリック
  2. 変更されたファイルが一覧表示される
  3. 各ファイルの「+」ボタンでステージング
  4. コミットメッセージを入力：「削除機能付きToDoアプリ v3.0を追加」
  5. 「コミット」ボタンをクリック
📸 スクリーンショット: VS Code Git操作画面

ステップ2: ブランチをGitHubにプッシュ

user@hostname:~/projects/js_tutorial$ git push origin feature/enhanced-todo # ブランチをGitHubにプッシュ

✅ チェックポイント

  • □ 変更がコミットされた
  • □ ブランチがGitHubにプッシュされた
  • □ GitHubでブランチが確認できる

3.4 プルリクエストの作成

プロの開発現場で使われるコードレビュー機能を体験します。

プルリクエストの作成

  1. ブラウザでGitHubリポジトリを開く
  2. 「Compare & pull request」ボタンが表示されるのでクリック
  3. タイトル：「削除機能付きToDoアプリ v3.0」
  4. 説明文を記入：
## 変更内容
- タスクの削除機能を追加
- UIの改善（削除ボタン）
- コードの関数化とモジュール化

## テスト方法
1. ToDoを追加
2. 削除ボタンをクリック
3. タスクが削除されることを確認

## レビューポイント
- 削除機能の動作確認
- UIの見た目確認
  1. 「Create pull request」をクリック
💡 重要: プルリクエストは単なるマージ手段ではなく、品質保証とナレッジ共有の仕組みです。実際のチームではここでコードレビューが行われます。

✅ チェックポイント

  • □ プルリクエストが作成された
  • □ 変更差分が表示されている
  • □ 説明文が記入されている

🤝 Phase 4: コラボレーション機能

⏱️ 所要時間： 約30分

4.1 プルリクエストのマージ

自分でレビューしてマージする体験をします。

セルフレビューとマージ

  1. 作成したプルリクエストページで「Files changed」タブをクリック
  2. 変更内容を確認（実際のレビューではここで他の人がコメント）
  3. 「Review changes」→「Approve」→「Submit review」
  4. 「Merge pull request」→「Confirm merge」
⚠️ 実際のチームでは: 自分でマージするのではなく、他のメンバーがレビューしてからマージします。

✅ チェックポイント

  • □ プルリクエストがマージされた
  • □ mainブランチに変更が反映された
  • □ 機能ブランチが削除された

4.2 ローカル環境の同期

リモートの変更をローカルに反映します。

user@hostname:~/projects/js_tutorial$ git checkout main
user@hostname:~/projects/js_tutorial$ git pull origin main
# リモートの最新状態をローカルに反映

user@hostname:~/projects/js_tutorial$ git branch -d feature/enhanced-todo
# 不要になった機能ブランチを削除

✅ チェックポイント

  • □ mainブランチに切り替えた
  • □ 最新の変更が反映された
  • □ 機能ブランチが削除された

4.3 リリース管理の実践

v3.0として新しいリリースを作成します。

GitHubでのリリース作成

  1. ブラウザでリポジトリの「Releases」を開く
  2. 「Create a new release」をクリック
  3. タグ名：v3.0
  4. タイトル：「ToDoアプリ v3.0 - 削除機能搭載版」
  5. 説明文：
## 新機能
- ✅ タスクの削除機能
- ✅ 改善されたUI/UX
- ✅ モジュール化されたコード

## アップグレード方法
1. script_v3.js を使用
2. 削除ボタンでタスクを削除可能

## 既知の課題
- ローカルストレージ未対応（次バージョンで対応予定）
  1. 「Publish release」をクリック

✅ チェックポイント

  • □ v3.0のリリースが作成された
  • □ リリースノートが記載された
  • □ zipファイルがダウンロード可能
