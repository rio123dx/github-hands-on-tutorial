🤔 現在の状況を振り返ってみましょう

現在、KintoneのJavaScriptを共有ファイルで管理されていますね。

❌ 共有ファイル管理の課題

  • 誤って上書きしてしまうリスク
    → 他の人が編集中に気づかず上書き
  • 過去のバージョンに戻せない
    → 「昨日まで動いていたのに...」が解決できない
  • 複数人での同時編集ができない
    → 誰かが編集中は待つしかない
  • 変更履歴や理由が残らない
    → 「なぜこの変更をしたのか」が分からない

💡 GitHubとは

GitHubは、Microsoft社が運営する世界最大の開発プラットフォームです。

  • Git（分散型バージョン管理システム）をベースとしたクラウドサービス
  • ソースコードの履歴管理、共同開発、コードレビューを効率化
  • 世界中の開発者が利用（Microsoft、Google、Appleも使用）

✅ GitHubで解決できること

  • 世代管理（バージョン管理）
    → 変更履歴を完全に保持、いつでも過去の状態に戻せる
  • 変更追跡
    → 誰が、いつ、何を、なぜ変更したかが記録される
  • ブランチ機能
    → 本番環境に影響せずに実験的な開発が可能
  • プルリクエスト（PR）
    → コードレビューによる品質向上
  • タグによるリリース管理
    → 安定版のスナップショットを作成・配布
  • Issue管理
    → バグ報告や機能要望の追跡

📊 比較：現在 vs GitHub

現在の共有ファイル

  • ❌ ファイル名で世代管理
    script_v1.js, script_v2.js...
  • ❌ 変更者・理由が不明
  • ❌ 同時編集でコンフリクト
  • ❌ バックアップは手動
  • ❌ コードレビューなし

GitHubでの管理

  • ✅ 自動的な履歴管理
    全ての変更が自動記録
  • ✅ 変更者・日時・理由が記録
  • ✅ 並行開発をマージで統合
  • ✅ クラウドで自動バックアップ
  • ✅ プルリクエストでレビュー
