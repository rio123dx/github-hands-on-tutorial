ステップ1: GitHubアカウント作成

まずはGitHubのアカウントを作成しましょう。

  1. ブラウザで github.com にアクセス
  2. 右上の 「Sign up」 ボタンをクリック
  3. 以下の情報を入力：
    • Username: 半角英数字で任意の名前
    • Email: 普段使うメールアドレス
    • Password: 8文字以上の安全なパスワード
📸 スクリーンショット: GitHubサインアップページ
⚠️ 注意: Usernameは後から変更できますが、URLに使われるので覚えやすいものにしましょう。

✅ チェックポイント

  • □ GitHubにログインできた
  • □ ダッシュボードページが表示された

ステップ2: 新規リポジトリの作成

コードを保存する「リポジトリ」を作成します。

  1. ダッシュボードの左側にある 「New」 ボタンをクリック
  2. リポジトリ名に js_tutorial と入力
  3. 「Public」 を選択（無料プランの場合）
  4. 「Add a README file」 にチェック
  5. 「Create repository」 ボタンをクリック
📸 スクリーンショット: リポジトリ作成画面

✅ チェックポイント

  • □ js_tutorial リポジトリが作成された
  • □ README.mdファイルが表示されている
  • □ URLが github.com/[ユーザー名]/js_tutorial になっている

ステップ3: README.mdの編集

プロジェクトの説明文を書いてみましょう。

  1. README.md ファイルをクリック
  2. 右上の鉛筆アイコン（✏️）をクリックして編集モードに
  3. 以下の内容に置き換え：
# Kintone JavaScript Tutorial

このリポジトリは、KintoneのJavaScript開発を学習するためのものです。

## ファイル構成
- index.html: メインのHTMLファイル
- script.js: ToDoアプリ v1.0
- script_v2.js: ToDoアプリ v2.0（機能拡張版）

## 学習内容
- GitHubの基本的な使い方
- バージョン管理の重要性
- コラボレーション開発
  1. 下部の 「Commit changes」 をクリック
  2. コミットメッセージに「プロジェクト説明を追加」と入力
  3. 「Commit changes」 ボタンをクリック

✅ チェックポイント

  • □ README.mdが更新された
  • □ ファイル履歴に「プロジェクト説明を追加」が記録された

ステップ6: script.jsの機能拡張（v2.0への更新）

既存のscript.jsファイルを更新して、入力欄と追加ボタンを実装します。

  1. script.js ファイルをクリック
  2. 右上の鉛筆アイコン（✏️）をクリックして編集モードに
  3. ファイル内容を以下のように完全に置き換え：
/**
* ToDoアプリ v2.0 - 機能拡張版
*/

// DOM要素を取得
const taskList = document.getElementById('taskList');
const addButton = document.getElementById('addButton'); // 追加ボタン
const taskInput = document.getElementById('taskInput'); // 入力欄

// 初期データ
const initialTasks = [
'洗濯をする',
'掃除をする',
'買い物に行く'
];

// 初期データを表示
initialTasks.forEach(taskText => {
const listItem = document.createElement('li');
listItem.textContent = taskText;
taskList.appendChild(listItem);
});

// ボタンクリック時の処理
addButton.addEventListener('click', () => {
const newTaskText = taskInput.value.trim();

if (newTaskText !== '') {
const listItem = document.createElement('li');
listItem.textContent = newTaskText;
taskList.appendChild(listItem);

// 入力欄をクリア
taskInput.value = '';
taskInput.focus();
}
});

// Enterキーでも追加できるように
taskInput.addEventListener('keypress', (e) => {
if (e.key === 'Enter') {
addButton.click();
}
});
  1. コミットメッセージに「ToDoアプリ v2.0 - 入力機能とボタン処理を追加」と入力
  2. 「Commit changes」 をクリック
💡 バージョン管理の実感: 同じファイルを更新することで、v1.0から v2.0への変更が明確に記録されます。ファイル履歴を見ると「どの部分が」「いつ」「なぜ」変更されたかが一目瞭然になります。これが共有ファイルでは不可能だった「変更履歴の追跡」です。

✅ チェックポイント

  • □ script.jsファイルが更新された
  • □ コミット履歴で v1.0 → v2.0 の変更が追跡されている
  • □ ファイルの変更差分（diff）が確認できる

ステップ7: プルリクエストでレビュー体験

チーム開発で重要な「レビュー機能」を体験してみましょう。タスク名を少し修正します。

  1. script.js ファイルをクリック
  2. 右上の鉛筆アイコン（✏️）をクリックして編集モードに
  3. 初期データの1行目を以下のように修正：
// 修正前
'洗濯をする',

// 修正後
'洗濯物を干す',
  1. 右上の 「Commit changes」 をクリック
  2. 表示されたダイアログで以下を設定：
    • コミットメッセージ：「タスク名を具体的に修正」
    • 「Create a new branch for this commit and start a pull request」 にチェック
    • ブランチ名を feature/update-task-names に変更
  3. 「Propose changes」 をクリック
💡 ブランチ名のベストプラクティス: feature/機能名 や fix/修正内容 のように、目的が分かる名前にするのが一般的です。
  1. 「Open a pull request」画面で以下を入力：
    • タイトル: 「タスク名をより具体的な表現に修正」
    • 説明: 「洗濯をする → 洗濯物を干す に変更して、より具体的なタスク内容にしました。」
  2. 「Create pull request」 をクリック
💡 レビューのポイント:
  • 「Files changed」タブで変更差分を確認
  • 緑色（+）が追加、赤色（-）が削除
  • 小さな変更でも、意図が伝わるか確認することが重要
  • 実際のチーム開発では、他の人がレビューしてからマージします
  1. 「Files changed」 タブで変更内容を確認
  2. 「Conversation」 タブに戻る
  3. 「Merge pull request」 をクリック
  4. 「Confirm merge」 をクリック
  5. 「Delete branch」 をクリックしてブランチを削除

✅ チェックポイント

  • □ プルリクエストが作成された
  • □ 変更差分が確認できた
  • □ プルリクエストがマージされた
  • □ mainブランチに変更が反映された

ステップ8: v1.0のリリース作成

安定版としてv1.0をリリースします。まず過去のコミットからリリースを作成します。

  1. 右側のサイドバーから 「Releases」 をクリック
  2. 「Create a new release」 をクリック
  3. タグ名に v1.0 と入力
  4. 「Choose a tag」 で「プロジェクト説明を追加」のコミットを選択
  5. リリースタイトルに「ToDoアプリ v1.0 - 初回リリース」と入力
  6. 説明に「基本的なHTML構造と初期データ表示機能」と入力
  7. 「Publish release」 をクリック

✅ チェックポイント

  • □ v1.0のリリースが作成された
  • □ ダウンロード可能なzipファイルが生成された

ステップ9: v2.0のリリース作成

機能拡張版をv2.0としてリリースします。最新のコミットからリリースを作成します。

  1. 「Create a new release」 をクリック
  2. タグ名に v2.0 と入力
  3. リリースタイトルに「ToDoアプリ v2.0 - 機能拡張版」と入力
  4. 説明に「入力機能とボタン処理を追加。ユーザーが新しいタスクを追加できるようになりました。」と入力
  5. 「Publish release」 をクリック

✅ チェックポイント

  • □ v2.0のリリースが作成された
  • □ 2つのバージョンがリリース一覧に表示された
